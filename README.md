# Groundwork

**Define your schema once. Get consistent AI queries every time.**

Groundwork is a developer tool for vibecoders who are tired of AI making up column names, inventing tables that don't exist, and writing queries that break because every chat session starts from scratch.

You describe your database in plain English. Groundwork parses it, shows you the structured result, and generates a **GROUNDWORK.md** context file you can drop into any AI coding session -- Claude, ChatGPT, Cursor, Copilot, whatever. From that point on, every query the AI writes uses your actual schema.

No more `user.firstName` when your column is `first_name`. No more `SELECT * FROM subscriptions` when the table is called `plans`. One file. Consistent output. Every time.

---

## How It Works

Groundwork is a 3-step flow:

**1. Describe** -- Paste your database schema in plain English. No SQL required.

> "I have users with name, email, and role. Posts belong to users and have a title, body, and published flag. Subscriptions track which plan each user is on."

**2. Review** -- Groundwork parses your description into structured tables with types, constraints, and relationships. Review them as visual cards before proceeding.

**3. Export** -- Copy or download your generated `GROUNDWORK.md` file. Drop it into your next AI session and start writing queries that actually work.

## Quick Start

```bash
# Clone the repo
git clone https://github.com/your-username/groundwork.git
cd groundwork

# Set up environment
cp .env.example .env.local
# Add your Anthropic API key to .env.local

# Install and run
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and paste a schema description to get started.

> You need an [Anthropic API key](https://console.anthropic.com/) to parse schemas. The key stays on your server and is never sent to the browser.

## Example Output

Here is what a generated `GROUNDWORK.md` looks like (trimmed for brevity):

```markdown
# GROUNDWORK.md
# Generated by Groundwork

## Schema: SaaS Application
A SaaS platform with user accounts, subscription billing, and a content publishing system.

---

## Tables

### users
Core user accounts for the platform.

| Column        | Type      | Constraints              |
|---------------|-----------|--------------------------|
| id            | integer   | primary key              |
| name          | string    |                          |
| email         | string    | unique                   |
| password_hash | string    |                          |
| role          | string    | default: user            |
| created_at    | timestamp | default: now()           |

### posts
User-generated content entries.

| Column     | Type      | Constraints              |
|------------|-----------|--------------------------|
| id         | integer   | primary key              |
| user_id    | integer   | fk -> users.id           |
| title      | string    |                          |
| body       | text      |                          |
| published  | boolean   | default: false           |
| created_at | timestamp | default: now()           |

## AI Instructions
...
### Example Queries

**posts**
-- Get posts with author info
SELECT t.*, r.*
FROM posts t
JOIN users r ON t.user_id = r.id
WHERE t.id = ?;
```

See [`examples/GROUNDWORK.md`](examples/GROUNDWORK.md) for a complete, fully generated example.

## Project Structure

```
src/
  app/
    page.tsx                  # Main page -- stepper state + step rendering
    layout.tsx                # Root layout with Inter font
    globals.css               # Tailwind + custom CSS variables
    api/
      parse-schema/route.ts   # POST -- sends input to Claude, returns schema
      generate-context/route.ts # POST -- builds GROUNDWORK.md from schema
  components/
    SchemaInput.tsx           # Step 1: textarea + parse button
    SchemaReview.tsx          # Step 2: table cards + relationships
    ContextExport.tsx         # Step 3: code block + copy/download
    StepSidebar.tsx           # Left sidebar step indicator
  lib/
    types.ts                  # Schema, Table, Column, Relationship types
    schema-validator.ts       # Zod schemas for Claude response validation
    context-generator.ts      # Deterministic GROUNDWORK.md builder
```
